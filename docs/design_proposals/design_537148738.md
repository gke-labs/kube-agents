## 1. Executive Summary

This technical design proposal establishes a comprehensive security hardening and automated governance framework for the `kube-agents` CI/CD infrastructure on GitHub Actions. By addressing findings PIPE-001 through PIPE-006, this design implements immutable action SHA pinning, safe output variable encapsulation, multi-layered static security analysis (`gosec`, `trivy`, `semgrep`, `checkov`), full-stack Dependabot ecosystem coverage (Go, Python, Node.js, Docker, Actions), and strict least-privilege workflow permissions.

---

## 2. Security Findings & Detailed Assessment

### 2.1 PIPE-001: Unpinned Third-Party GitHub Actions

- **Current State:** Workflows throughout `.github/workflows/` reference third-party GitHub Actions via floating branch or tag identifiers (e.g., `uses: actions/checkout@v7`, `uses: docker/build-push-action@v7`, `uses: actions/setup-go@v7`).
- **Security Risk:** Mutable tag references can be updated or overwritten upstream (either intentionally or via account compromise), introducing supply chain attack vectors or unexpected breaking changes into build pipelines.
- **Target Fix:** Pin every action invocation to an immutable 40-character git commit SHA, accompanied by an inline comment indicating the release tag (e.g., `uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2`).

### 2.2 PIPE-002: Insecure `$GITHUB_OUTPUT` Writes & Unverified Binary Downloads

- **Current State:**
  - In `docker-publish-gcp.yml` and `reusable-deploy-integrations.yml`, dynamic step outputs are written directly using single-line string concatenation (`echo "digest=$DIGEST" >> "$GITHUB_OUTPUT"`).
  - In `actionlint.yml`, binary dependencies are retrieved via unverified `curl -sSfL` downloads without hash verification.
- **Security Risk:**
  - Direct string concatenation without heredoc delimiters can lead to output truncation, multiline line-injection, or broken workflow evaluation if output strings contain spaces, newlines, or control characters.
  - Downloading unverified binaries via `curl` introduces remote code execution and supply chain risks.
- **Target Fix:** Standardize output writes to use safe heredoc delimiter syntax (`{ echo "key<<EOF"; echo "$VAL"; echo "EOF"; } >> "$GITHUB_OUTPUT"`) and replace raw `curl` scripts with official, pinned GitHub Actions (`rhysd/actionlint-action`).

### 2.3 PIPE-003: Missing Static Security Analysis Scanners

- **Current State:** CI pipelines lack automated static security scanners for Go source code, container images, Infrastructure-as-Code (Kubernetes manifests, Helm charts, Dockerfiles), and Python scripts.
- **Security Risk:** Vulnerabilities such as hardcoded credentials, permissive Kubernetes RBAC/Pod security policies, outdated container bases, or unsafe code patterns can bypass review into production.
- **Target Fix:** Implement a dedicated, multi-job `.github/workflows/security-scan.yml` incorporating `gosec` (Go SAST), `trivy` (container & manifest misconfiguration scan), `semgrep` (multi-language SAST), and `checkov` (IaC compliance).

### 2.4 PIPE-005: Incomplete Dependabot Ecosystem Coverage

- **Current State:** `.github/dependabot.yml` monitors only `github-actions` at the root and `docker` for subdirectories `/agents/platform`, `/agents/operator`, `/agents/devteam`. It lacks tracking for Go (`k8s-operator/go.mod`), Python (`tests/e2e/requirements.txt`), Node.js (`package.json`), and additional Docker directories (`deploy/docker`).
- **Security Risk:** Outdated language packages and dependencies are left unmonitored for known CVEs.
- **Target Fix:** Expand `.github/dependabot.yml` to include `gomod`, `pip`, `npm`, and additional `docker` directory updates on a scheduled weekly cadence.

### 2.5 PIPE-006: Insecure Event Triggers & Permissive Permissions Hygiene

- **Current State:** Certain workflows lack top-level `permissions:` blocks or require review for dangerous triggers (e.g. `pull_request_target`).
- **Security Risk:** Workflows executed without strict permission boundaries can be exploited to modify repository state or leak credentials.
- **Target Fix:** Enforce top-level `permissions: contents: read` across all workflow files, restricting elevated permissions (`id-token: write`, `pull-requests: write`) exclusively to necessary jobs.

### 3.2 Output Sanitization & Safe Scripting (PIPE-002)

1. Update `docker-publish-gcp.yml` output writes:

   ```bash
   # Before:
   # echo "digest=$DIGEST" >> "$GITHUB_OUTPUT"

   # After:
   {
     echo "digest<<EOF"
     echo "$DIGEST"
     echo "EOF"
   } >> "$GITHUB_OUTPUT"
   ```

2. Replace actionlint script downloading in `.github/workflows/actionlint.yml`:
   ```yaml
   name: Actionlint

   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]

   permissions:
     contents: read

   jobs:
     actionlint:
       runs-on: ubuntu-latest
       steps:
         - name: Checkout code
           uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2

         - name: Run actionlint
           uses: rhysd/actionlint-action@486e9cd4ae9876778f654b9d08e5033734eb60e1 # v1.7.7
   ```

### 3.3 Automated Security Scanning Workflow (PIPE-003)

Create `.github/workflows/security-scan.yml` with four dedicated jobs:

```yaml
name: Security Scanning

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
  schedule:
    - cron: "0 4 * * 1" # Weekly Monday scan

permissions:
  contents: read

jobs:
  gosec-scan:
    name: Go Security Scan (gosec)
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2
      - name: Run gosec
        uses: securego/gosec@master
        with:
          args: "./k8s-operator/..."

  trivy-scan:
    name: Container & Manifest Scan (trivy)
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2
      - name: Scan Repository Configs & Manifests
        uses: aquasecurity/trivy-action@master
        with:
          scan-type: "config"
          exit-code: "1"
          severity: "HIGH,CRITICAL"

  semgrep-scan:
    name: SAST Scan (semgrep)
    runs-on: ubuntu-latest
    container:
      image: semgrep/semgrep
    steps:
      - name: Checkout code
        uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2
      - name: Run semgrep
        run: semgrep ci --config=p/security-audit

  checkov-scan:
    name: IaC Security Scan (checkov)
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2
      - name: Run Checkov
        uses: bridgecrewio/checkov-action@master
        with:
          framework: all
          output_format: cli
          soft_fail: false
```

### 3.5 Permissions Scoping & Event Trigger Security (PIPE-006)

- Ensure all workflow files set explicit top-level `permissions: contents: read`.
- Restrict job-level `id-token: write` and `pull-requests: write` exclusively to steps requiring GCP Workload Identity Federation or GitHub API comment/review creation.

## 4. Code Comments & Potential Conflicts Analysis

A comprehensive audit of the codebase (`.github/workflows/*`, `.github/dependabot.yml`, `Makefile`, and `k8s-operator/`) was conducted to evaluate potential conflicts:

1. **No Incompatible Directives:** No inline comments or documentation directives were found that require mutable action tag references or block static security scanners.
2. **Actionlint Version Alignment:** The current `actionlint.yml` workflow installs version `1.7.7`. By replacing the custom `curl` script with `rhysd/actionlint-action@486e9cd4ae9876778f654b9d08e5033734eb60e1` (# v1.7.7), we preserve exact version compatibility while eliminating raw script downloads.
3. **Prettier Integration:** Prettier is used in `prettier.yml` and called via `npx prettier --check` in `Makefile`. Adding `npm` package ecosystem tracking to `.github/dependabot.yml` ensures lockfiles and prettier versions remain safely updated without breaking existing lint targets.
